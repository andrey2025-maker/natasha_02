# Cargo_Omsk55 Bot Architecture (Phase 1)

## Цель
Разделить систему на маленькие независимые модули, чтобы Telegram и VK-часть развивались параллельно через общую доменную логику и БД.

## Текущие слои
- `app/domain`: сущности и перечисления без привязки к платформе.
- `app/services`: бизнес-правила (коды профиля, синхронизация, антиспам).
- `app/security`: подпись и проверка callback-данных.
- `app/storage`: интерфейсы репозиториев (для PostgreSQL реализации).
- `app/bot`: будущие адаптеры Telegram/VK.
- `app/main.py`: точка сборки зависимостей.

## Раздельный запуск по серверам
- Telegram сервер: `python -m bots.telegram_bot.main`
- VK сервер: `python -m bots.vk_bot.main`
- Оба процесса используют общий PostgreSQL (`DATABASE_URL`) как единое хранилище состояния и настроек.
- Для локальной разработки можно оставлять комбинированный раннер: `python -m app.main`.

## Уже реализовано из ТЗ
1. Подпись callback-кнопок HMAC-SHA256 (`CallbackSigner`).
2. Базовые anti-spam лимиты и отсечка размеров сообщений/медиа (`RateLimitService`).
3. Базовый сервис кодов профиля и ограничение работы с резервом (`ProfileCodeService`).
4. Сервис синхронизации профилей VK/TG с TTL, cooldown и антибрут ограничением (`SyncService`).
5. PostgreSQL-схема и async-репозитории (`app/storage/postgres/*`).
6. In-memory репозитории для локальной разработки (`DATABASE_URL=memory`).
7. FSM заполнения профиля (`ProfileFlowService`) + Telegram handlers `/start`, «Профиль», «Заполнить профиль».
8. Подписанные callback-кнопки с привязкой к `user_id` (`CallbackCodec`).
9. Синхронизация профиля VK↔TG в профильном сценарии (`SYNC_ENTER_CODE`, `SYNC_VERIFY`).
10. Сценарий «Заказ выкупа» с сохранением заявок (`BuyoutFlowService` + `BuyoutOrderRepository`).
11. Каркас VK-адаптера (`app/bot/vk/*`) для дальнейшего подключения SDK.
12. Outbox-механизм исходящих сообщений в платформы (`outbound_messages`) для кросс-платформенной доставки кодов.
13. Раздел «Мои заказы» в Telegram с пагинацией и новыми заказами сверху.
14. Фоновый VK outbox worker (`run_vk_outbox_worker`) с доставкой `sync_code` через VK API.
15. Входящий VK runtime (`run_vk_bot`) с командами `/start`, «Профиль», «Заполнить профиль», «Есть профиль ТГ», «Заказ выкупа», «Мои заказы».
16. Фильтры статусов в «Мои заказы» с сохранением в сессии (`_prefs.order_filters`) для Telegram и VK.
17. FAQ-модуль: древовидные разделы (`faq_sections`), админское управление и пользовательская навигация в Telegram/VK.
18. Обновление статусов заказов менеджерами + журнал изменений (`buyout_order_status_history`) и вывод истории в «Мои заказы».
19. Админ UI массового обновления статусов заказов (чекбоксы, пакетная смена статуса, пагинация).
20. Редактирование одного выбранного заказа в админке (ссылка, количество, цена, комментарий, трек).
21. Блок «Статистика» в админке: агрегаты пользователей, статусы заказов и недельная динамика регистраций.
22. Массовое редактирование полей заказов (цена, комментарий, трек, детали) для выбранных заказов.
